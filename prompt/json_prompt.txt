You will be given structured extracts derived from a source document.

Your task is to produce two artifacts from this information:

1. A JSON representation that structurally captures selected information from the document.
2. A JSON Schema (Draft 2020-12) that describes exactly and only the structure of the generated JSON.

The JSON must be fully derivable from the document text alone.

---

## GENERAL CONSTRAINTS

* Do NOT introduce facts, interpretations, inferred categories, reorganizations, or summaries.
* Every value included in the JSON must be explicitly grounded in the document.
* If information is not explicitly stated, omit it entirely.
* Do not fabricate values.

---

## KEY RULES (MANDATORY)

1. ALL JSON keys MUST be in English using snake_case.
   * Translate document field names to natural English (not literal word-for-word).
   * Keys do NOT need to appear in the document — descriptive English keys are allowed and preferred.
   * Proper nouns (names, places) used as keys may be romanized if no standard English exists.

2. Values MUST remain as extracted from the document (original language preserved).

3. If the same concept appears multiple times, consolidate all information under a single key.

---

## STRUCTURE RULES

* Use arrays for ordered or repeated items (e.g., rows in a table, list of entries).
* Use objects when items have named fields or a clear key-value structure.
* Structure must be deterministic.
  Independent implementations must produce identical JSON from the same document.
* Do not introduce structural grouping not explicitly supported by the document.
* Preserve the explicit hierarchy as presented.

---

## VALUE STANDARDIZATION RULES (CRITICAL – STRICT LEXICAL CONSTRAINT)

All values MUST obey the following lexical constraint:

1. A value must be a single atomic word OR a numeric value.
2. A value must NOT be a sentence.
3. A value must NOT be a clause.
4. A value must NOT contain explanatory phrasing.
5. A value must NOT contain commas, periods, conjunctions, or connective expressions.
6. A value must NOT describe, summarize, interpret, or elaborate.
7. A value must NOT restate the source text in prose form.
8. A value must NOT contain multiple descriptive words forming a sentence-like structure.

If a value cannot be expressed as:

* a single word (lexical token), OR
* a pure number

THEN:
You MUST omit that key entirely.

Do NOT attempt to summarize.
Do NOT compress a sentence into a short phrase.
Do NOT approximate.

If strict lexical atomicity is not possible, remove the key.

Numeric rules:

* If a numeric value includes units, return only the numeric core.
  “2023년” → 2023
  “15%” → 15
  “3년 과정” → 3

* Do NOT include unit words.

* Do NOT include explanatory modifiers.

* Do NOT include null, "", 0, false, or placeholders unless explicitly stated.

---

## JSON SCHEMA REQUIREMENTS

* Use JSON Schema Draft 2020-12.
* The schema must describe exactly and only the JSON you generate.
* Do not include any properties not present in the JSON.
* Field types must exactly match the JSON types.
* Reflect optional fields accurately using "required".
* If object maps are used, use "additionalProperties" to define their value structure.
* The schema must allow reconstruction of the JSON without ambiguity.
* Do NOT include commentary in the schema.
* Every object and property definition MUST include a "title" field (the key name or entity name as a string).
* Every property definition MUST include a "description" field that briefly explains what the field represents in the context of the document.

---

## SELF-VALIDATION BEFORE OUTPUT (MANDATORY)

Before finalizing the response, internally verify:

1. The output contains exactly two sections in the required order.
2. The first section header is exactly: === JSON ===
3. The second section header is exactly: === JSON_SCHEMA ===
4. There is no text before the first header.
5. There is no text after the schema.
6. No Markdown or code fences exist.
7. All keys are in English snake_case.
8. Arrays are used for ordered/repeated items; objects for named fields.
9. No inferred data has been introduced.
10. Every value is either:

    * a single atomic word, OR
    * a pure number
11. No value contains sentence structure.
12. If any value required explanation, the key has been removed.

Only after validation, produce the final output.

---

## OUTPUT FORMAT (STRICT)

The output MUST consist of exactly two top-level sections:

=== JSON === <valid JSON object>

=== JSON_SCHEMA ===
<valid JSON Schema Draft 2020-12 object>

Rules:

* Do not wrap in code fences.
* Do not indent headers.
* Do not add blank lines before the first line.
* Do not add commentary.
* Do not add trailing text.
* Output must begin with "=== JSON ==="
